## ADDED Requirements

### Requirement: 按主机配置报告发送
系统 SHALL 允许用户为单台主机开启/关闭报告发送，并使用 cron 选择器字段（`type`、`where1`、`hour`、`minute`、`week`）配置频率。

#### Scenario: 开启报告发送
- **WHEN** 用户为主机开启报告发送
- **THEN** 该主机的发送状态被持久化为已开启
- **AND** 当前频率配置被保存

#### Scenario: 关闭报告发送
- **WHEN** 用户为主机关闭报告发送
- **THEN** 该主机的发送状态被持久化为已关闭
- **AND** 频率配置入口在 UI 中隐藏

### Requirement: 主机列表启用标识
系统 SHALL 在主机列表的报告概览旁显示邮件图标，用于提示该主机已开启报告发送。

#### Scenario: 列表启用标识
- **WHEN** 主机列表渲染已开启报告发送的主机
- **THEN** 报告概览旁显示邮件图标

### Requirement: 按频率发送主机报告邮件
系统 SHALL 按主机配置的频率发送该主机的报告内容到邮箱，并使用面板/PVE 对应的 HTML 模板渲染邮件内容，前端展示与邮件发送共用同一套模板。

#### Scenario: 定时触发发送
- **WHEN** 当前时间匹配主机配置的频率
- **THEN** 系统发送包含该主机报告内容的邮件
- **AND** 记录发送时间以避免同一时间窗重复发送

#### Scenario: 渲染 HTML 报告内容
- **WHEN** 系统准备发送主机报告邮件
- **THEN** 使用对应类型（面板或 PVE）的 HTML 模板渲染报告内容
- **AND** 模板从独立的模板文件加载

#### Scenario: 邮件配置缺失
- **WHEN** 定时触发发送但邮件通知未配置
- **THEN** 系统记录失败并跳过发送，调度器不中断

#### Scenario: 复用现有邮件通知配置
- **WHEN** 系统发送主机报告邮件
- **THEN** 发送行为复用现有邮件通知配置的发送方与收件人
