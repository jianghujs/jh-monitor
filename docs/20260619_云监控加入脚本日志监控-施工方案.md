# 云监控加入脚本日志监控施工方案

## 1. 方案目标

把防火墙同步、主备 NAS 同步等脚本日志统一收集到 ES，由云监控按任务频率和执行状态进行分析，并在每日报告中输出正常、告警、异常结果。

## 2. 总体流程

```text
被监控主机执行初始化命令，自动注册监控项到云监控
        │
        ▼
被监控主机执行云监控生成的初始化命令
        │
        ▼
初始化 filebeat、日志路径、jh-monitor-task-log、监控任务配置
        │
        ▼
业务脚本调用 jh-monitor-task-log 写入标准化 JSON 日志
        │
        ▼
filebeat 将任务日志采集到 ES：host-monitor-task-event
        │
        ▼
云监控按频率和状态分析任务最新状态
        │
        ▼
每日报告读取“监控任务”和 ES 数据生成分析报告
```

## 3. 监控任务模块

云监控新增“监控任务”模块，用于定义监控项、频率、状态和安装信息。

建议表名：`monitor_task`

| 字段 | 说明 |
|---|---|
| `id` | 自增 ID |
| `task_id` | 任务唯一 ID，由云监控生成 |
| `task_name` | 监控项名称，例如“防火墙同步”“主备 NAS 同步” |
| `host_id` | 归属主机 ID |
| `host_name` | 归属主机名称，冗余展示 |
| `host_ip` | 归属主机 IP，冗余展示 |
| `log_path` | 被监控主机上的任务日志路径 |
| `check_interval` | 检查频率，建议单位为秒 |
| `enabled` | 是否启用 |
| `install_status` | 安装状态：`pending/installed/failed/unknown` |
| `last_status` | 最新分析状态：`normal/warning/error/unknown` |
| `last_msg` | 最新任务消息 |
| `last_run_at` | 最新任务日志时间 |
| `addtime` | 创建时间 |
| `update_time` | 更新时间 |

第一版最小字段：

```text
task_id
task_name
host_id
log_path
check_interval
enabled
```

建议同时保留：

```text
last_status
last_msg
last_run_at
install_status
```

## 4. 初始化命令

所有监控项的添加和更新都通过被监控主机执行初始化命令完成。命令自动完成 filebeat 采集配置、`jh-monitor-task-log` 命令下发、监控任务注册等流程。

示例命令：

```bash
wget -O /tmp/install_monitor_task.sh http://<monitor-host>:10844/pub/get_monitor_task_install_script \
  && bash /tmp/install_monitor_task.sh install \
    --server-url http://<monitor-host>:10844 \
    --task-id MT_nas_sync_1234 \
    --task-name "主备 NAS 同步" \
    --log-path /var/log/jh-monitor/tasks/nas_sync.json.log \
    --check-interval 86400
```

参数说明：

| 参数 | 说明 |
|---|---|
| `--server-url` | 云监控服务地址 |
| `--task-id` | 任务 ID |
| `--task-name` | 监控项名称 |
| `--log-path` | 任务日志路径 |
| `--check-interval` | 检查频率，建议单位为秒 |
| `--host-id` | 可选，不传则读取本机已有 host_id |
| `--run-user` | 可选，业务脚本运行用户，用于日志目录授权 |

脚本职责：

```text
1. 读取本机 host_id，优先复用 /home/ansible_user/jh-monitor-data/host_id
2. 在云监控中添加或更新 monitor_task
3. 创建任务日志目录和日志文件
4. 配置 filebeat 采集指定日志文件
5. 下发 jh-monitor-task-log 命令
6. 校验并 reload/restart filebeat
7. 回写 install_status 到云监控
```

## 5. 标准日志命令

业务脚本统一通过 `jh-monitor-task-log` 写入任务结果日志，不直接拼接 JSON。

最小用法：

```bash
jh-monitor-task-log --task-id "MT_nas_sync_1234"
```

常用示例：

```bash
jh-monitor-task-log --task-id "MT_nas_sync_1234" --status success --msg "同步完成"
jh-monitor-task-log --task-id "MT_nas_sync_1234" --status warning --msg "同步耗时超过 10 分钟"
jh-monitor-task-log --task-id "MT_nas_sync_1234" --status error --msg "同步失败：rsync exit 23"
```

参数规则：

| 参数 | 是否必填 | 默认值 | 说明 |
|---|---|---|---|
| `--task-id` | 是 | 无 | 监控项 ID |
| `--status` | 否 | `success` | 只允许 `success/warning/error` |
| `--msg` | 否 | 按状态生成 | 结果摘要 |
| `--run-at` | 否 | 当前时间 | 任务运行或写入时间 |

`--msg` 默认值：

| status | 默认 msg |
|---|---|
| `success` | `成功` |
| `warning` | `告警` |
| `error` | `异常` |

下发路径：

```text
/home/ansible_user/jh-monitor-scripts/bin/jh-monitor-task-log
/home/ansible_user/jh-monitor-scripts/lib/jh_monitor_task.py
/usr/local/bin/jh-monitor-task-log -> /home/ansible_user/jh-monitor-scripts/bin/jh-monitor-task-log
```

本地任务配置：

```text
/home/ansible_user/jh-monitor-tasks/<task_id>.json
```

配置示例：

```json
{
  "task_id": "MT_nas_sync_1234",
  "task_name": "主备 NAS 同步",
  "host_id": "H_nas_1234",
  "log_path": "/var/log/jh-monitor/tasks/nas_sync.json.log"
}
```

## 6. 日志格式

任务日志采用 JSON Lines，每一行是一条任务结果。

最小格式：

```json
{"status":"success","msg":"成功"}
```

推荐格式：

```json
{"status":"success","msg":"同步成功，变更 3 个文件","run_at":"2026-06-19 02:00:00"}
```

状态说明：

| status | 含义 | 云监控状态 |
|---|---|---|
| `success` | 任务正常 | 正常 |
| `warning` | 任务有提醒，但不一定失败 | 告警 |
| `error` | 任务失败或严重异常 | 异常 |

如果业务脚本未写入 `run_at`，分析时使用 ES 文档时间或 filebeat `@timestamp` 兜底。

## 7. filebeat 配置

filebeat 使用可组合配置，避免新增监控项时覆盖已有采集配置。

配置结构：

```text
/etc/filebeat/filebeat.yml
/etc/filebeat/inputs.d/jh-monitor-task-<task_id>.yml
```

主配置启用 `inputs.d`：

```yaml
filebeat.config.inputs:
  enabled: true
  path: /etc/filebeat/inputs.d/*.yml
  reload.enabled: true
  reload.period: 10s
```

单个监控项 input 示例：

```yaml
- type: log
  enabled: true
  paths:
    - /var/log/jh-monitor/tasks/nas_sync.json.log
  json.keys_under_root: true
  json.add_error_key: true
  json.overwrite_keys: true
  fields:
    log_index: host-monitor-task-event
    task_id: MT_nas_sync_1234
    task_name: 主备 NAS 同步
    host_id: H_nas_1234
  fields_under_root: true
```

## 8. ES 索引

所有监控任务日志统一写入：

```text
host-monitor-task-event
```

第一版不按任务拆索引，避免每新增一种脚本都要新增查询逻辑。

核心字段：

| 字段 | 类型建议 | 说明 |
|---|---|---|
| `task_id` | keyword | 任务唯一 ID |
| `task_name` | keyword/text | 任务名称 |
| `host_id` | keyword | 主机 ID |
| `host_ip` | ip | 主机 IP |
| `status` | keyword | `success/warning/error` |
| `msg` | text | 任务结果消息 |
| `run_at` | date | 任务运行或产生日志时间 |
| `log_path` | keyword | 日志路径 |
| `collector_source` | keyword | 固定为 `filebeat` 或 `monitor_task` |

ES 文档示例：

```json
{
  "task_id": "MT_nas_sync_1234",
  "task_name": "主备 NAS 同步",
  "host_id": "H_nas_1234",
  "host_ip": "192.168.1.20",
  "status": "success",
  "msg": "同步完成",
  "run_at": "2026-06-19 02:00:00",
  "log_path": "/var/log/jh-monitor/tasks/nas_sync.json.log",
  "collector_source": "filebeat"
}
```

查询规则：

```text
按 task_id + host_id 过滤
按 run_at 或 @timestamp 倒序
取最近一条任务日志
```

## 9. 状态判定

云监控按 `check_interval` 和最近一条 ES 日志判断任务状态。

判定输入：

```text
task_id
host_id
check_interval
最近一条 ES 事件
当前分析时间 now
```

判定公式：

```text
last_event = 最近一条任务日志
last_run_at = last_event.run_at 或 @timestamp
deadline = last_run_at + check_interval
```

第一版规则：

```text
如果没有任何日志：
  状态=error，msg=未发现任务日志

如果 now > last_run_at + check_interval：
  状态=error，msg=未按照频率产生日志 + 最后一条 msg

如果 last_event.status == success：
  状态=normal，msg=最后一条 msg

如果 last_event.status == warning：
  状态=warning，msg=最后一条 msg

如果 last_event.status == error：
  状态=error，msg=最后一条 msg

其他情况：
  状态=unknown 或 error，msg=未知 status + 最后一条 msg
```

状态优先级：

```text
异常 > 告警 > 正常
```

建议字段上预留：

```text
grace_seconds
```

第一版可默认 `grace_seconds=0`，后续用于减少执行时间波动导致的误报。

## 10. 后端分析任务

新增定时任务：`monitorTaskAnalyseTask`

职责：

```text
1. 读取 enabled=true 的 monitor_task
2. 按 task_id + host_id 查询 ES 最近一条日志
3. 按 check_interval 和 status 判定 normal/warning/error
4. 更新 monitor_task.last_status、last_msg、last_run_at
5. 为每日报告提供监控任务状态数据
```

流程：

```text
读取启用监控项
        │
        ▼
查询 ES 最近日志
        │
        ▼
按频率和 status 判定状态
        │
        ▼
更新 monitor_task 最新状态
        │
        ▼
日报读取最新状态并汇总
```

## 11. 日报接入

每日报告新增“监控项报告”。

单机报告展示：

```text
监控项：
┌──────────────┬────────┬────────────────────────┐
│ 监控项名称      │ 状态     │ 最近消息                 │
├──────────────┼────────┼────────────────────────┤
│ 防火墙同步      │ 正常     │ 同步成功                 │
│ 主备 NAS 同步   │ 异常     │ 同步失败：rsync exit 23   │
└──────────────┴────────┴────────────────────────┘
```

单机报告规则：

| 条件 | 处理 |
|---|---|
| 任一监控项异常 | 单机报告 `is_abnormal=true`，加入 `error_tips` |
| 任一监控项告警且无异常 | 单机报告 `is_warning=true`，加入提醒摘要 |
| 全部监控项正常 | 展示明细，不影响异常状态 |

总览报告增加异常摘要：

```text
监控项异常摘要：
- nas-master（192.168.1.20）：主备 NAS 同步异常，同步失败：rsync exit 23
- gateway（192.168.1.10）：防火墙同步异常，超过 86400 秒未产生日志，最后消息：同步成功
```
