# 报告数据收集、分析、发送流程优化方案

## 背景

- 发送报告只能通过江湖面板发送，并且一个服务器一个服务器发送，容易被识别为垃圾邮件。
- 数据收集、数据分析、消息发送耦合在一起，容易受当时服务器运行状态、SSH 连通性和瞬时异常影响。
- 数据目前主要保存在面板本地 sqlite 中，没有统一沉淀到 ES 做集中管理、查询和追溯。

## 总体思路

将现有流程拆成三段：

1. 数据收集：远端主机定时生成原始数据，并统一汇总到 ES。
2. 数据分析：暂时只扫描当天的原始数据，生成当天的日报，包括每个主机的报告和全部主机的概览报告。
3. 数据发送：从 ES 读取已经分析完成且校验通过的报告，再统一发送。

这样做的好处是：

- 采集、分析、发送解耦，链路更稳定。
- 报告以周期数据为准，不受发送当时机器状态影响。
- 原始数据和最终报告都能留痕，方便追溯、补算和重发。

## 一、报告数据收集

### 目标

定时到各主机采集报告所需的原始数据，统一写入 ES，作为后续分析的数据来源。

### 方案

- 在`江湖云监控`增加独立的采集任务，按固定频率执行。
- 通过 ansible 远程执行采集脚本，让目标主机本地生成报告的原始数据。
- 原始数据只负责记录事实，不直接拼装最终报告。
- 原始数据通过日志采集或接口方式统一进入 ES。

### 采集内容
- 主机信息（主机名、主机IP、主机组、主机状态）
- 采集时间
- 系统资源（CPU、内存、资源使用率、磁盘）
- 备份
  - MySQL主从 同步状态
  - xtrabackup 备份情况
  - xtrabackup增量 备份情况
  - rsync 同步情况
  - MySQL Dump 备份情况
- 网站状态（域名、证书、状态）
- JianghuJS项目状态（项目名、状态）
- Docker项目状态（项目名、状态）
- MySQL数据库状态（变化值、总大小）

## 二、报告数据分析

### 目标

将当天的原始数据汇总分析，生成当天的日报，并写入 ES 的报告索引。报告分为两类：每个主机的详细报告，以及全部主机的概览报告。

### 方案

- 在`江湖云监控`增加独立的分析任务，每天凌晨1点执行。
- 从 ES 中读取某台主机当天的原始数据。
- 按统一规则进行汇总、异常识别、摘要生成。
- 先生成每个主机的详细报告，再汇总生成全部主机的概览报告。
- 生成最终报告后写入 ES 报告索引，作为发送的唯一数据来源。

### 产出内容

- 单主机报告：展示某台主机当天的资源、备份、网站和服务状态。
- 主机概览报告：汇总全部主机当天的整体情况，重点展示异常主机、异常项分布和整体运行摘要。

## 三、报告数据发送

### 目标

从 ES 中读取当天已经生成且有效的报告，再统一发送，避免发送时临时去主机上取数和计算。

发送范围控制为：只发送异常服务器的单主机报告，以及全部主机的概览报告。

### 方案

- 在江湖云监控增加独立的发送任务，按报告配置定时执行。
- 发送时只从 ES 报告索引读取已经分析完成的报告。
- 单主机报告只发送当天识别为异常的服务器。
- 每天固定发送一份全部主机的概览报告。
- 对发送前的数据做时间和完整性检查。
- 发送成功后记录发送状态，失败时记录失败原因并支持重试。
